# Senior Engineer Advisor v4.1

**オンデマンド・アーキテクチャ指導によるコスト効率型AIコーディング**

実装エージェントに先に試行させ、行き詰まった時だけ専門家にコンサル。成功パターンをLLM Wikiに蓄積して再利用する、トークン効率の哲学に基づくアプローチ。

---

## 🎯 哲学

従来のアプローチ: *"先に全部設計してから実装"*

**私たちのアプローチ**: *"先に実装してみて、行き詰まったら専門家に相談。ナレッジを蓄積する"*

この哲学は以下の観察に基づいています：
- **現代の中級モデル（GLM-5）**は多くのタスクを自律的にこなせる
- **フロンティアモデルの相談**は、実装エージェントが本当に行き詰まった時に最も価値がある
- **ナレッジ蓄積**はLLM Wikiを通じて複利効果を生む

---

## ✨ 主な機能

### 1. オンデマンド・アドバイザー（v4.1）
```bash
advisor --on-demand --auto "タスク"
```

**フロー**:
1. 実装エージェント（GLM-5）が先にタスクを試行
2. 自己評価（複雑度/自信）
3. **行き詰まった時のみ**（複雑度 >= 0.5）: アドバイザー+専門家にコンサル
4. ガイダンスを受けて継続
5. 成功パターンをWikiに保存

### 2. 動的専門家選択
**226名のエージェント**からタスク分析に基づき3-8名を選択：
- キーワードベースのヒューリスティック選択（高速・確実）
- 不要なコンサルを排除
- kilo失敗時はopencode CLIにフォールバック

### 3. アドバイス深度設定（5段階）
```bash
advisor --depth 4 --auto "複雑なタスク"
```

| レベル | 詳細度 | 使用場面 |
|--------|--------|----------|
| 1 | シンプル - 大枠のみ | ルーチンタスク |
| 2 | 一般的 - 主要決定事項 | 標準的機能 |
| 3 | **標準** - アーキテクチャ+落とし穴+ベストプラクティス | **デフォルト** |
| 4 | 詳細 - 実装提案含む | 複雑システム |
| 5 | 包括的 - 徹底分析 | 重要アーキテクチャ |

### 4. LLM Wikiナレッジサイクル
```
タスク → Wiki検索（類似度 >= 0.75） → 実装 → Wiki保存 → 再利用
```

**利点**:
- 1回目: フルコンサル
- 2回目以降: パターン再利用（87%コスト削減）
- Usage Historyで継続的改善

## 🚀 クイックスタート

### インストール
```bash
# リポジトリをクローン
git clone https://github.com/JunSuzuki1973/senior-engineer-advisor.git
cd senior-engineer-advisor

# シンボリックリンク作成
ln -sf $(pwd)/integrations/advisor.sh ~/.local/bin/advisor

# 環境設定
export WIKI_DIR="$HOME/openclaw-wiki"
export AA_DIR="$HOME/agency-agents"
export PATH="$HOME/.local/bin:$PATH"
```

### 使用例

```bash
# 推奨: オンデマンドモード（賢明な相談）
advisor --on-demand --auto "JWT認証システムを構築"

# 標準: 複雑度自動判定
advisor --auto "Reactコンポーネントを作成"

# 実験用: 強制アドバイザー
advisor --force --depth 5 --auto "マイクロサービス設計"

# 条件付き: Wikiに知識がない場合のみ
advisor --wiki-only --auto "新技術を調査"

# プレビューモード
advisor --dry-run --auto "タスク"
```

## ⚙️ 設定

### 環境変数
```bash
# 必須
export WIKI_DIR="$HOME/openclaw-wiki"              # LLM Wikiパス
export AA_DIR="$HOME/agency-agents"                # Agency Agentsパス

# オプション
export ADVICE_DEPTH="3"                            # デフォルト深度（1-5）
export DEFAULT_MODEL="opencode-go/glm-5"          # 実装モデル
export ADVISOR_MODEL="kilo/anthropic/claude-opus-4-6"  # アドバイザーモデル
```

### 設定ファイル（config.yaml）
```yaml
advisor:
  general:
    model: "kilo/anthropic/claude-opus-4-6"
    max_tokens: 800
  advice_depth: 3                    # 1-5
  
  code_review:
    enabled: true
    require_permission: true

knowledge:
  llm_wiki:
    enabled: true
    path: "~/openclaw-wiki"
    auto_save: true
```
